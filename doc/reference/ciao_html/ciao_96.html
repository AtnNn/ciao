Pure Prolog package

Author(s): The CLIP Group.

Version: 1.11#120 (2003/12/22, 23:35:35 CET)

Version of last change: 1.11#100 (2003/12/21, 20:16:33 CET)

This library package allows the use of pure Prolog in a Ciao module/program. It is based on the fact that if an engine module is imported explicitly then all of them have to be imported explicitly. The engine modules are:

  • engine(arithmetic) section Arithmetic.
  • engine(atomic_basic) section Basic predicates handling names of constants.
  • engine(attributes) section Attributed variables.
  • engine(basic_props) section Basic data types and properties.
  • engine(basiccontrol) @xref{Control constructs/predicates}.
  • engine(data_facts) section Fast/concurrent update of facts.
  • engine(exceptions) @xref{Exception handling}.
  • engine(io_aux) section Message printing primitives.
  • engine(io_basic) section Basic input/output.
  • engine(prolog_flags) section Changing system behaviour and various flags.
  • engine(streams_basic) section Basic file/stream handling.
  • engine(system_info) section Gathering some basic internal info.
  • engine(term_basic) section Basic term manipulation.
  • engine(term_compare) section Comparing terms.
  • engine(term_typing) section Extra-logical properties for typing.

Note that if any of these modules is explicitely imported in a program then the language defaults to Pure Prolog, plus the functionality added by the modules explicitely imported.

It is recommended that if you explicitely import an engine module you also use this package, which will guarantee that the predicate true/0 is defined (note that this is the only Ciao builtin which cannot be redefined).

Usage and interface (pure)

  • Library usage: :- use_package(pure). or :- module(...,...,[pure]).

Known bugs and planned improvements (pure)

  • Currently, the following builtin predicates/program constructs cannot be redefined, in addition to true/0: (->)/2 (,)/2 (+)/1 if/3
