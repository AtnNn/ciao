Extra-logical properties for typing

Author(s): Daniel Cabeza, Manuel Hermenegildo.

Version: 1.11#143 (2003/12/31, 16:44:44 CET)

Version of last change: 1.11#124 (2003/12/26, 20:6:44 CET)

This library contains traditional Prolog predicates for testing types. They depend on the state of instantiation of their arguments, thus being of extra-logical nature.

  • Usage and interface (term_typing)
  • Documentation on exports (term_typing)

Usage and interface (term_typing)

  • Library usage: These predicates are builtin in Ciao, so nothing special has to be done to use them.
  • Exports:
    • Predicates: var/1, nonvar/1, atom/1, integer/1, float/1, atomic/1, ground/1, type/2.
    • Properties: var/1, nonvar/1, atom/1, integer/1, float/1, number/1, atomic/1, ground/1, type/2.

Documentation on exports (term_typing)

PROPERTY: number/1:

Usage 1: number(X)

  • Description: X is currently instantiated to a number.
  • The following properties hold globally: This predicate is understood natively by CiaoPP. (basic_props:native/1)

Usage 2: number(X)

  • The following properties hold upon exit: X is a number. (basic_props:num/1)
  • The following properties hold globally: X is not further instantiated. (basic_props:not_further_inst/2) This predicate is understood natively by CiaoPP. (basic_props:native/1)

Usage 3: number(X)

  • Description: X is currently instantiated to a number.

  • The following properties hold globally: X is not further instantiated. (basic_props:not_further_inst/2) This predicate is understood natively by CiaoPP. (basic_props:native/1)
