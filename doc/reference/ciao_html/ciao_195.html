Documentation on exports (generator)

PREDICATE: generator/2:

Usage 1: generator(+Terms,-VRML)

  • Description: This predicate is the generator of VRML code. It accepts a list of terms that is correct VRML code, other kind of terms will be rejected will errormessage accordingly. The output is a string of correct VRML code, acceptable for VRML browsers.
  • Call and exit should be compatible with: +Terms is a list of termss. (basic_props:list/2) -VRML is a string (a list of character codes). (basic_props:string/1)

Usage 2: generator(Terms,VRML)

  • Description: This predicate is the generator of VRML code. It accepts a list of terms that is correct VRML code, other kind of terms will be rejected will errormessage accordingly. The output is a string of correct VRML code, acceptable for VRML browsers.
  • Call and exit should be compatible with: Terms is a list of termss. (basic_props:list/2) VRML is a string (a list of character codes). (basic_props:string/1)
  • The following properties should hold at call time: Terms is currently a term which is not a free variable. (term_typing:nonvar/1) VRML is a free variable. (term_typing:var/1)
