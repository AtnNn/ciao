Functional notation

Version: 1.9#93 (2003/7/29, 17:53:15 CEST)

Version of last change: 1.7#186 (2002/2/4, 19:11:45 CET)

This library package allows the use of functional notation in a Ciao module/program.

All function applications (definitions) are translated to goals (predicate definitions) where another argument to hold the result of the function is added to the right. Function applications need to be preceded by the operator ~, or the functor be declared as such by using the declaration function/1. There is an exception: all functions understood by is/2 are considered as functions by default. This feature can be disabled by a declaration :- function(arith(false)) (and reverted by using true instead of false). A functor normally considered as a function call can be escaped using the prefix operator ^. Function definitions can be written by using the binary operator :=, and can have also body. Example of use: 

:- use_package(functions).

fib(0) := 0.
fib(1) := 1.
fib(N) := ~fib(N-1) + ~fib(N-2) :- integer(N), N > 1.

write_fib(N):-
        message(['The ',N,'. Fibonacci number is: ',~fib(N),'.']).
